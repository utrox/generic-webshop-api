Recovery page

Normally, the link you've got in the email would point to the front-end where the user can fill out a form to set their new password. Because this project has no front-end, I've decided to implement this simple page as a temporary solution, so that the password-recovery functionality can be tested. You can also chose to make a POST request with Postman:

POST request: localhost:3000/api/v1/auth/recovery/<token> JSON body: {"newPassword": "<newPassword>", "confirmPassword": "<newPassword>"}
